Introduction

The Bicycle Parking Lot Occupancy Locator (VéloParking) is a web application designed to provide real-time information about the availability of bicycle parking spaces in various locations. This system aims to help cyclists find available parking spots easily and efficiently, thereby promoting the use of bicycles as a sustainable mode of transportation.

System Architecture

The system architecture of VéloParking is divided into three main tiers: the Presentation Tier, the Application Tier, and the Data Tier.

Presentation Tier (User Interface)

This tier is responsible for the user interface and user experience. It includes the web pages and mobile interfaces that users interact with to access the system’s features.

Application Tier (Backend Services)

The application tier handles the business logic of the system. It includes the server-side applications and services that process user requests, manage data flow, and ensure the system’s functionalities are executed correctly.

Data Tier (Database)

The data tier is where all the data is stored and managed. This includes user information, parking lot details, occupancy data, and historical records. The database ensures data integrity, security, and availability.

Presentation Tier Design (WireFrame)

The presentation tier design is typically represented through wireframes that outline the structure and layout of the user interface. These wireframes serve as blueprints for the actual UI development.
